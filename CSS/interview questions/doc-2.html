It is used to specify the starting and ending positions of an item within the grid's columns. - **Syntax**: `grid-column: start / end;` - `start`: The starting column (from the left). - `end`: The ending column (from the left, excluding it). #### Example: ```css .item { grid-column: 1 / 3; /* Spans from column 1 to 3 */ } ``` This means the item will occupy columns 1 and 2. --- ### **grid-row** Defines the vertical placement of a grid item. It specifies the starting and ending positions of an item within the grid's rows. - **Syntax**: `grid-row: start / end;` - `start`: The starting row (from the top). - `end`: The ending row (from the top, excluding it). #### Example: ```css .item { grid-row: 2 / 4; /* Spans from row 2 to 4 */ } ``` This means the item will occupy rows 2 and 3. --- ### **Examples** #### Example 1: Placing Items Using `grid-column` and `grid-row` ```css .grid-container { display: grid; grid-template-columns: 100px 1fr; grid-template-rows: 50px 100px 150px; } .item1 { grid-column: 1 / 2; /* Starts at column 1, ends at column 2 */ grid-row: 1 / 3; /* Starts at row 1, ends at row 3 */ } .item2 { grid-column: 2 / 3; /* Starts at column 2, ends at column 3 */ grid-row: 2 / 4; /* Starts at row 2, ends at row 4 */ } ``` Here: - `.item1` spans the first column and rows 1 to 2. - `.item2` starts at the second column and spans rows 2 to 3. --- ### **Shorthand for Spanning Across Multiple Columns/Rows** You can also use shorthand to define how many rows or columns the item should span. - **`grid-column: span 2;`**: This spans the item across 2 columns. - **`grid-row: span 3;`**: This spans the item across 3 rows. #### Example: ```css .item { grid-column: span 2; /* Spans 2 columns */ grid-row: span 2; /* Spans 2 rows */ } ``` --- In summary: - **`grid-column`** controls horizontal positioning (columns). - **`grid-row`** controls vertical positioning (rows). It can be moved using `top`, `right`, `bottom`, and `left`, but still occupies space in the layout. - **Absolute Positioning**: The element is removed from the document flow and positioned relative to the nearest positioned ancestor (an ancestor with `position: relative`, `absolute`, or `fixed`). Elements that form a stacking context are ordered by their `z-index` values. Each stacking context contains elements with their own stacking order. - The `z-index` property determines the order in which elements stack in a stacking context. Elements with a higher `z-index` value appear in front of those with lower values. Only elements within the same stacking context are affected by `z-index`. 2. **How does position: sticky behave differently from fixed?** - **Position: sticky**: The element toggles between relative and fixed positioning. It behaves like a relatively positioned element until the page scrolls past a specified threshold, at which point it becomes fixed and stays in place as the user scrolls. - **Position: fixed**: The element is fixed relative to the viewport and remains in the same position even when the page is scrolled. It doesn't depend on the page scroll and stays in place at all times. --- ### **Display: Flex** #### **Basic:** 1. **What is the difference between justify-content and align-items?** - **justify-content**: Aligns items along the main axis (horizontal by default, vertical if `flex-direction: column`). - **align-items**: Aligns items along the cross axis (vertical by default, horizontal if `flex-direction: column`). 2. **How does flex-direction: column affect layout?** - `flex-direction: column` changes the main axis from horizontal to vertical, making the flex container stack its items vertically instead of horizontally.
